## Layer 1: Data Ingestion (Prompt 1)

Real-time data pipeline ingesting commodity prices, geopolitical events, and supply chain status.

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ Price Feeds │  │ Geo Events  │  │ Supply Data │
│  (Lambda)   │  │  (Lambda)   │  │  (Lambda)   │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │                │                │
       ▼                ▼                ▼
┌─────────────────────────────────────────────┐
│           EventBridge Custom Bus             │
│     "minescope-data-bus"                    │
│  ┌────────────┐ ┌──────────┐ ┌───────────┐ │
│  │price-ticks │ │geo-high  │ │supply-    │ │
│  │→Kinesis    │ │→SQS+SNS  │ │updates    │ │
│  │            │ │→Kinesis  │ │→Kinesis   │ │
│  └────────────┘ └──────────┘ └───────────┘ │
└────────────────┬────────────────────────────┘
                 │
       ┌─────────┼──────────┐
       ▼         ▼          ▼
┌──────────┐ ┌────────┐ ┌─────────┐
│ Price    │ │ Event  │ │ Supply  │
│Aggregator│ │Processor│ │Processor│
│ (Lambda) │ │(Lambda)│ │ (Lambda)│
└────┬─────┘ └───┬────┘ └────┬────┘
     │           │           │
     ▼           ▼           ▼
┌─────────────────────────────────┐
│   S3 Raw Data Lake              │
│   + DynamoDB Tables             │
│   + EventBridge (alerts)        │
└─────────────────────────────────┘
```

**Key Components:**
- **3 Ingestion Lambdas**: Simulate price tick data (5 minerals, every 5-30s), geopolitical events (1-5/min), and facility status (every 30-60s)
- **EventBridge Routing**: 5 rules routing events by type and severity to Kinesis, SQS, and SNS
- **3 Processor Lambdas**: Aggregate OHLCV candles, enrich geopolitical events, track facility status
- **3 DynamoDB Tables**: latest-prices (TTL: 72h), risk-scores, facility-status — all with GSIs
- **S3 Data Lake**: Partitioned raw storage by mineral/date/event-type

## Layer 2: AI/ML (Prompt 2)

NLP pipeline using Claude 3.5 Sonnet via Bedrock for geopolitical risk analysis.

```
┌──────────────┐
│ News Articles│
│ (S3 Ingest)  │
└──────┬───────┘
       ▼
┌──────────────┐     ┌──────────────┐
│ Article      │────▶│ Translation  │
│ Parser       │     │ (Translate)  │
│ (Lambda)     │     └──────┬───────┘
└──────┬───────┘            │
       │                    │
       ▼                    ▼
┌──────────────┐     ┌──────────────┐
│ Language     │────▶│ Analysis     │
│ Detection    │     │ Queue (SQS)  │
│ (Comprehend) │     └──────┬───────┘
└──────┬───────┘            │
       │                    │
       ▼                    ▼
┌──────────────┐     ┌──────────────────────┐
│ Entity       │────▶│ Claude Risk Analyzer │
│ Extractor    │     │ (Bedrock → Claude    │
│ (Comprehend) │     │  3.5 Sonnet)         │
└──────────────┘     └──────┬───────────────┘
                            │
                    ┌───────┼───────┐
                    ▼       ▼       ▼
              ┌──────────┐┌──────┐┌──────────┐
              │ Risk     ││Alert ││ Daily    │
              │ Scores   ││Gen.  ││ Briefing │
              │(DynamoDB)││(SNS) ││ (S3+Bedr.)│
              └──────────┘└──────┘└──────────┘
```

**Key Components:**
- **Amazon Comprehend**: Custom NER for minerals, mining companies, government bodies, trade policies
- **Amazon Translate**: Multi-language news processing (Chinese, Spanish, French, Portuguese sources)
- **Claude 3.5 Sonnet**: Expert system prompt generates structured risk assessments with severity, supply impact %, price impact %, confidence, reasoning, and recommended actions
- **Bedrock Guardrails**: Content filtering, topic restriction, PII redaction
- **Human-in-the-loop**: Severity ≥ 8 assessments flagged for manual review before applying to risk scores
- **Daily Intelligence Briefing**: AI-generated Markdown report distributed via SNS

## Layer 4: Data Plane (Prompts 1, 3, 6)

Unified data storage with multi-region replication.

| Data Store | Primary Region | Replication | Purpose |
|-----------|---------------|-------------|---------|
| S3 Raw Data | us-east-1 | CRR → eu-west-1, ap-southeast-1 | Immutable event storage |
| DynamoDB Global Tables | us-east-1 | Active-active → eu-west-1, ap-southeast-1 | Latest prices, risk scores, facility status |
| S3 Analytics | eu-west-1 | CRR from us-east-1 | Athena query data |
| S3 Briefings | us-east-1 | CRR → eu-west-1 | Daily AI intelligence reports |
| S3 FinOps | us-east-1 | No replication | Cost and usage reports |

## Layer 6: IaC, DevOps & FinOps (Prompts 5, 7)

Complete deployment automation and cost governance.

```
GitHub Push
    │
    ▼
CodePipeline (CI/CD)
    ├── Build: npm install, tsc, lint, test
    ├── Security: cdk-nag, checkov
    ├── Deploy Dev: CDK deploy → sandbox
    ├── Integration Tests
    ├── Manual Approval (Slack)
    ├── Deploy Prod: CDK deploy → us-east-1
    ├── Deploy DR: CDK deploy → eu-west-1, ap-southeast-1
    └── Smoke Tests + Notify

CDK Stack Dependency:
Security → Pipeline → Bedrock → Analytics → FinOps → DR → Monitoring
```
